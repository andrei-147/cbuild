## đźš€ The Concept

`cbuild` operates on a simple principle: **If it's in the folder and has a valid extension, it gets compiled.** It recursively crawls your project directory to identify source files and automatically includes parent directories for headers.

## âś¨ Features

* **Zero Config:** No `.json`, `.toml`, or `Makefile` required.
* **Auto-Inclusion:** Automatically detects header directories.
* **Recursive Discovery:** Finds source files in nested subdirectories.
* **Exclusion Rules:** Automatically ignores "hidden" files or private folders starting with `.` or `_`.
* **Compiler Flexibility:** Defaults to `g++` but supports any compiler passed as an argument.

## đź’» Usage

### Basic Build

By default, `cbuild` generates an executable named `exec`.

```bash
# Basic debug build using g++
./cbuild debug

# Specify a different compiler (e.g., clang++)
./cbuild release clang++

# Specify the C / C++ standard
./cbuild release g++ c++23

```

### Command Arguments

1. **Mode:** `debug` or `release`.
2. **Compiler (Optional):** Define which compiler to invoke. Defaults to `g++`.
3. **Standard (Optional but requires compiler):** Specify the C/C++ standard used when building the project. Defaults to `c++17`.

## đź› ď¸Ź How it works

1. **Cleanup:** Removes any existing `exec` binary to ensure a fresh build.
2. **Scan:** Recursively walks the current path.
3. **Filter:** Ignores directories and files starting with `.` or `_`.
4. **Map:** Source files are collected into a list.
* Unique header directories are collected into a set (no duplicate `-I` flags).


5. **Execute:** Constructs a system command and executes it:
`[compiler] [src_files...] -I[header_dirs...] -o exec [debug flags... | release flags...] -std=[c/c++xx]`
